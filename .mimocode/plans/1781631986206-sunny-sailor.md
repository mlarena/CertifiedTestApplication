# Исправления Testing/Question — логические ошибки и безопасность

## Критические проблемы (6)

### 1. CanReturnToQuestion не проверяется на сервере
**Файл**: `Controllers/TestingController.cs`, действие `Question`
Пользователь может перейти на любой вопрос через URL `?questionNumber=5`, даже если `CanReturnToQuestion=false`. Кнопки навигации по номерам тоже всегда кликабельны.
**Фикс**: В `Question` action проверять: если `CanReturnToQuestion==false` и `questionNumber` больше текущей позиции — редиректить на текущий/следующий неотвеченный.

### 2. Finish — GET без CSRF, без проверки таймаута
**Файл**: `Controllers/TestingController.cs`, действие `Finish`
- Finish это GET-запрос без `[ValidateAntiForgeryToken]` — CSRF-уязвимость
- Не проверяется таймаут — пользователь может обойти таймер
**Фикс**: Сделать `Finish` POST-действием с `[ValidateAntiForgeryToken]`. Добавить проверку таймаута. В Question.cshtml заменить ссылку на форму с POST.

### 3. SaveAnswer не проверяет принадлежность questionId к тесту
**Файл**: `Controllers/TestingController.cs`, действие `SaveAnswer`
Принимает любой `questionId` без проверки, что он принадлежит тесту из `attemptId`.
**Фикс**: `if (!await _context.Questions.AnyAsync(q => q.Id == questionId && q.TestId == attempt.TestId)) return BadRequest();`

### 4. SaveAnswer не проверяет принадлежность selectedAnswerIds к вопросу
**Файл**: `Controllers/TestingController.cs`, действие `SaveAnswer`
GUID-ы ответов вставляются без проверки.
**Фикс**: Фильтровать `selectedAnswers` — оставить только те, что принадлежат `questionId`.

### 5. Сравнение double == double для числовых ответов
**Файл**: `Controllers/TestingController.cs`, строка ~242
`val == correctAnswer.NumericValue` — некорректно из-за плавающей запятой.
**Фикс**: `Math.Abs(val - (correctAnswer.NumericValue ?? 0)) < 0.01` (допуск 0.01).

### 6. Start теста с 0 вопросов
**Файл**: `Controllers/TestingController.cs`, действие `Start`
Тест без вопросов создаёт попытку и сразу редиректит на Finish с 0%.
**Фикс**: Проверять `questions.Count > 0` перед стартом.

## Средние проблемы (5)

### 7. saveAndNavigate теряет ответы из-за 200ms setTimeout
**Файл**: `Views/Testing/Question.cshtml`
`saveAnswer()` — async fetch, а навигация через фиксированный `setTimeout(200ms)`. При медленной сети ответ не успевает сохраниться.
**Фикс**: Сделать `saveAndNavigate` async, дожидаться завершения fetch перед навигацией.

### 8. Браузерная кнопка Back показывает кэшированную страницу
**Файл**: `Views/Testing/Question.cshtml`
Нет `Cache-Control: no-store`. Таймер и ответы устаревают.
**Фикс**: Добавить `pageshow` обработчик с `location.reload()` при `e.persisted`.

### 9. Повторное.resume попытки всегда на вопрос 1
**Файл**: `Controllers/TestingController.cs`, действие `Start`
При наличии незавершённой попытки — редирект на `questionNumber=1`.
**Фикс**: Найти первый неотвеченный вопрос и редиректить на него.

### 10. Дублирование Order у вопросов одного теста
**Файл**: `Models/Entities/Entities.cs`, `Data/ApplicationDbContext.cs`
Нет уникального ограничения на `(TestId, Order)`.
**Фикс**: Добавить составной уникальный индекс в `OnModelCreating`.

### 11. Блокировка пользователя не проверяется во время теста
**Файл**: `Controllers/TestingController.cs`
Если заблокировать пользователя во время прохождения — он продолжает тест.
**Фикс**: Проверять `IsBlocked` в `Question` и `SaveAnswer`.

## Низкоприоритетные (3)

### 12. `duration.Minutes` возвращает компонент (0-59), не общее количество минут
**Файл**: `Views/Testing/Finish.cshtml`
**Фикс**: `@((int)duration.TotalMinutes) мин @duration.Seconds сек`

### 13. Нет `ThenBy(q => q.Id)` как tiebreaker для сортировки вопросов
**Файл**: `Controllers/TestingController.cs`
**Фикс**: Добавить `.ThenBy(q => q.Id)` после `OrderBy(q => q.Order)`.

### 14. Timer drifft — серверная задержка не компенсируется
**Файл**: `Views/Testing/Question.cshtml`
**Фикс**: Вычесть 2-3 секунды из начального `secondsLeft`.
