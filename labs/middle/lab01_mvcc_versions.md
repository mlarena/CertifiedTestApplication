# Лабораторная работа №1: Многоверсионность (MVCC)

## Цель
Изучить механизм многоверсионности PostgreSQL: скрытые столбцы xmin/xmax, видимость версий строк, поведение при параллельном доступе.

## Теория
MVCC (Multiversion Concurrency Control) — механизм, позволяющий читающим и пишущим транзакциям работать параллельно без взаимных блокировок. PostgreSQL хранит несколько версий одной строки. Каждая версия помечена номерами транзакций xmin (создавшая) и xmax (удалившая или обновившая).

## Задание

### Шаг 1. Подготовка

```sql
CREATE DATABASE lab_mvcc;
\c lab_mvcc

CREATE TABLE t(s text);
INSERT INTO t VALUES ('Версия 1');
```

### Шаг 2. Наблюдение за xmin/xmax

```sql
-- Текущая транзакция
BEGIN;
SELECT pg_current_xact_id();

-- Видимость версий
SELECT *, xmin, xmax FROM t;
```

### Шаг 3. Параллельное изменение

```sql
-- Сеанс 1
BEGIN;
SELECT *, xmin, xmax FROM t;
-- xmin=735, xmax=0 (актуальная версия)

-- Сеанс 2
BEGIN;
UPDATE t SET s = 'Версия 2';
COMMIT;

-- Сеанс 1
SELECT *, xmin, xmax FROM t;
-- xmax показывает, что строка изменена другой транзакцией

COMMIT;
SELECT *, xmin, xmax FROM t;
-- Теперь видна Версия 2
```

### Шаг 4. Блокировки при изменении одной строки

```sql
-- Сеанс 1
BEGIN;
UPDATE t SET s = 'Версия 3' RETURNING *;

-- Сеанс 2
BEGIN;
UPDATE t SET s = 'Версия 4';  -- Ждёт!

-- Сеанс 1
COMMIT;

-- Сеанс 2
-- Теперь выполняется
COMMIT;
```

### Шаг 5. Фиксация и откат

```sql
-- Сеанс 1
BEGIN;
UPDATE t SET s = 'Откатываемая';
-- Не коммитим

-- Сеанс 2
SELECT * FROM t;
-- Видит старое значение

-- Сеанс 1
ROLLBACK;

-- Сеанс 2
SELECT * FROM t;
-- Всё ещё видит старое значение
```

## Решение

### Шаг 2: Результат
```
BEGIN;
SELECT pg_current_xact_id();
 pg_current_xact_id
--------------------
                 736
(1 row)

SELECT *, xmin, xmax FROM t;
    s      | xmin | xmax
-----------+------+------
 Версия 1  |  735 |     0
(1 row)
```

## Контрольные вопросы

1. Что означает `xmax = 0`?
2. Почему читающие транзакции никогда не блокируют пишущие?
3. Как уменьшить количество «мертвых» версий строк?
