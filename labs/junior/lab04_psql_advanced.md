# Лабораторная работа №4: Продвинутый psql

## Цель
Изучить переменные psql, скрипты, взаимодействие с ОС и команды для работы с результатами запросов.

## Теория
psql поддерживает переменные (аналогичные переменным shell), условные операторы (`\if`), выполнение скриптов (`\i`), вывод в файлы (`\o`), а также выполнение команд операционной системы (`\!`). Переменные используются для переиспользования текстов запросов и хранения промежуточных результатов.

## Задание

### Шаг 1. Переменные psql

```sql
-- Установка переменной
\set myvar 'Hello PostgreSQL'

-- Вывод значения
\echo :myvar

-- Запись результата запроса в переменную
SELECT now() AS current_time \gset

-- Использование
\echo Текущее время: :current_time

-- Все переменные
\set

-- Удаление переменной
\unset myvar
```

### Шаг 2. Выполнение команд ОС

```sql
-- Команда shell
\! pwd

-- Переменная окружения
\setenv MY_TEST_VAR test_value
\! echo $MY_TEST_VAR
```

### Шаг 3. Вывод в файл

```sql
-- Перенаправление вывода в файл
\o /tmp/psql_output.txt

SELECT schemaname, tablename
FROM pg_tables
WHERE schemaname = 'pg_catalog'
LIMIT 5;

-- Вернуть вывод на экран
\o

-- Проверить содержимое файла
\! cat /tmp/psql_output.txt
```

### Шаг 4. Команда \gexec

```sql
-- Динамическое выполнение SQL из результата запроса
SELECT format('SELECT count(*) AS cnt FROM %I;', tablename)
FROM pg_tables
WHERE schemaname = 'pg_catalog'
LIMIT 3
\gexec
```

### Шаг 5. Скрипты

```sql
-- Сохраняем скрипт
\! echo "SELECT version();" > /tmp/test_script.sql
\! echo "SELECT current_database();" >> /tmp/test_script.sql

-- Выполняем скрипт
\i /tmp/test_script.sql
```

### Шаг 6. Условные конструкции

```sql
-- Проверка существования переменной
\echo :{?nonexistent_var}

-- Условный оператор
\if :{?nonexistent_var}
  \echo Переменная существует
\else
  \echo Переменная не существует
\endif
```
