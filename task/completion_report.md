# Отчет о реализации проекта АСТП

Проект «Автоматизированная система технического тестирования пользователей» 
 Ниже представлен список выполненных работ и ключевых функций.

## 1. Архитектура и База данных
- **Стек**: .NET 10.0, ASP.NET Core MVC, Entity Framework Core.
- **БД**: PostgreSQL (настроено подключение к серверу `192.168.19.129`).
- **Миграции**: Создана и применена начальная структура таблиц (Пользователи, Роли, Тесты, Вопросы, Ответы, Попытки, Логи).
- **Начальные данные**: Созданы роли (Admin, Engineer, User) и системный администратор (`sa` / `sa`).

## 2. Безопасность и Аутентификация
- **Своя система Auth**: Реализована аутентификация на Cookie без использования громоздкого Identity.
- **Хэширование**: Использование библиотеки `BCrypt.Net-Next` для безопасного хранения паролей.
- **Ролевая модель**: Доступ к разделам ограничен атрибутами `[Authorize(Roles = "...")]`.
- **Логирование**: 
  - Важные действия (создание тестов, блокировка пользователей) пишутся в таблицу `Logs` в БД.
  - Ошибки и системные события пишутся в файлы в папку `logs/` с использованием Serilog.

## 3. Функциональные возможности

### Режим «Администрирование»
- Управление пользователями (создание, блокировка).
- Управление категориями тестов (с защитой от удаления категорий с тестами).
- Просмотр системного журнала действий.

### Режим «Конструктор тестов» (Инженер)
- Создание тестов с настройкой тайминга.
- Редактор вопросов трех типов:
  1. Одиночный выбор.
  2. Множественный выбор.
  3. Числовой ввод (точное совпадение).
- Загрузка изображений к вопросам (хранятся в `wwwroot/uploads/questions/`).

### Режим «Тестирование» (Пользователь)
- Главная страница со списком доступных тестов.
- **Автосохранение**: Ответы сохраняются в БД мгновенно через AJAX при выборе.
- Навигация по вопросам и индикатор прогресса.
- Автоматический расчет результата (процент верных ответов).

### Режим «Результаты и отчеты»
- Личный кабинет пользователя со списком попыток.
- **Разбор ошибок**: Детальный просмотр пройденного теста с подсветкой верных/неверных ответов и выбором пользователя.
- Общий журнал тестирования для администрации.

## 4. Интерфейс (UI/UX)
- Использован светлый стиль **Bootstrap**.
- Применены компактные классы (`-sm`, `table-sm`) для удобства работы на десктопах и планшетах.
- Интерфейс полностью на русском языке.
