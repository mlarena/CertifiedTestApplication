Техническое задание на разработку программного обеспечения
«Автоматизированная система технического тестирования пользователей (АСТП)»

1. Цель и назначение разработки
Создать приложение для автоматизации процессов:
Создания, хранения и прохождения технических тестов.
Автоматической оценки результатов по критериям (процент
правильны\неправильных ответов).
Формирования протоколов тестирования с выводим правильный-неправильный
ответ.

Разграничения ролей: «Пользователь» (тестируемый), «Инженер по
тестированию» (автор), «Администратор».

2. Требования к функциональности
2.1. Режим «Администрирование»
Создание/редактирование категорий тестов (например: «Администрирование»,
«ТБ»,  «Знание 1С», «Техника безопасности»).

Управление пользователями (создание, блокировка, сброс пароля).

Назначение прав: кто может создавать тесты, кто может просматривать
результаты.

2.2. Режим «Конструктор тестов» (для инженеров)
Типы вопросов:
Одиночный выбор (радиокнопка).
Множественный выбор (check-box).
Числовой ввод (эталон с допуском, например 24.5).



Логика завершения: можно ли вернуться к вопросу? -да.

Тайминг: общий на тест или на каждый вопрос.

Привязка файлов: возможность вставить изображение.

2.3. Режим «Техническое тестирование» (для пользователя)
Авторизация по логину/паролю.

Отображение статуса: «Старт»,  «Номер вопроса».

Техническая надежность:

Автосохранение ответа после каждого шага (защита от вылета).

Завершение: После посленего вопроса, по кнопке завершить. Автоматический
подсчет баллов.

2.4. Режим «Результаты и отчеты»
Личный кабинет пользователя: видит свои оценки и разбор ошибок.

Кабинет инженера:

Журнал всех попыток (фильтр по группе, дате, статусу «Сдал/Не сдал»).


3. Технические требования (для разработчика)
3.1. Архитектура
Серверная часть Бэкенд: dotnet core c# asp.net core mvc javascript.

База данных: PostgreSQL (хранить вопросы, ответы, хэши сессий).


Фронтенд: asp.net core mvc javascript.


3.2. Производительность


3.3. Безопасность
Шифрование паролей.

Журналирование всех действий: кто, когда, какой тест.

Защита от SQL-инъекций, XSS.

4. Требования к интерфейсу (UI/UX)
Адаптивность: минимальное разрешение 1366x768 (десктоп) + планшеты.

стили bootrap. эмодзи запрежены.
Индикатор загрузки при сохранении ответа.
